# Официальный портал правовой информации Елогорской Республики

Статический сайт для официального опубликования нормативных правовых актов
вымышленной **Елогорской Республики**: указы, законы, постановления, обращения,
дипломатические ноты, международные договоры, ведомственные приказы и др.

Полностью статический — запускается без бэкенда, прямо на **GitHub Pages**.

![republic emblem](assets/emblem.svg)

---

## Возможности

### Публичная часть (`index.html`)

- Полный реестр опубликованных актов с виджетами:
  - **поиск** по ключевым словам (название, краткое описание, теги, автор, номер);
  - **фильтр по типу** (35+ предустановленных категорий — от поправок к
    Конституции до дипломатических нот и приказов министерств);
  - **фильтр по принявшему органу**;
  - **фильтр по диапазону дат**;
  - **сортировка** по дате (новые / старые), по названию (А→Я, Я→А), по типу.
- **Просмотрщик документов прямо в браузере**:
  - PDF — через [PDF.js](https://mozilla.github.io/pdf.js/);
  - DOCX — через [mammoth.js](https://github.com/mwilliamson/mammoth.js);
  - TXT — нативно.
- Печать любого документа из просмотрщика, скачивание оригинала.
- Стиль — поздние 2000-е / начало 2010-х: глянцевые градиенты, Aero-кнопки,
  бевели, тени, шапка с гербом — официально и узнаваемо.

### Закрытая часть — админ-панель (`bulbazavr-moldovan.html`)

- «Скрытый» URL — путь задаётся в `js/config.js`. По умолчанию
  `bulbazavr-moldovan.html`. Меняйте имя файла и `ADMIN_PATH` под себя.
- Простая форма входа, логин/пароль из `js/config.js`.
- Полное CRUD-управление актами:
  - публикация нового акта (название, дата, тип, номер, автор, описание, теги,
    файл — PDF/DOCX/TXT);
  - редактирование существующих;
  - переименование;
  - удаление.
- Drag-and-drop загрузки файла, валидация обязательных полей.
- Все правки сначала складываются в `localStorage` браузера админа.
- Когда правки готовы — кнопка **«Скачать архив для публикации»** генерирует
  `ZIP`, в котором лежит обновлённый `data/manifest.json` и все новые файлы
  актов в `data/acts/`. Достаточно распаковать архив в корень репозитория,
  закоммитить и сделать `git push` — GitHub Pages обновит сайт автоматически.

> **Важно про безопасность.** Это статический сайт, поэтому реальная серверная
> аутентификация невозможна. Учётные данные в `js/config.js` доступны клиенту;
> защита здесь — символическая (обфускация). Реальная защита публикации — это
> право `git push` в репозиторий. Это нормальный и распространённый подход
> для статических CMS.

## Локальный запуск

Сайт — чистая статика, но из-за `fetch()` ему нужен HTTP-сервер. Любой подойдёт:

```bash
# Python 3
python3 -m http.server 8000

# Node
npx http-server -p 8000

# PHP
php -S localhost:8000
```

Открыть в браузере: <http://localhost:8000/>.

Админка по умолчанию: <http://localhost:8000/bulbazavr-moldovan.html>.
Логин/пароль по умолчанию — см. `js/config.js` (`admin` / `elogor-2025`).
**Перед публикацией обязательно поменяйте.**

## Развёртывание на GitHub Pages

1. Создайте публичный репозиторий.
2. Закоммитьте всё содержимое проекта в корень `main` (или другой дефолтной ветки).
3. В настройках репозитория: **Settings → Pages**.
   - Source: `Deploy from a branch`
   - Branch: `main` / `(root)`
4. Сохраните. GitHub выдаст URL вида `https://<user>.github.io/<repo>/`.
5. Если используете кастомный домен (например, `pravo.elogor.gov.er`),
   укажите его в Pages-настройках и добавьте файл `CNAME` в корень.

> Файл `.nojekyll` в репозитории уже есть — он отключает обработку через Jekyll
> и нужен, чтобы GitHub Pages не игнорировал директории, начинающиеся с `_`.

## Как опубликовать новый акт (рабочий процесс)

1. Откройте `bulbazavr-moldovan.html` (логин / пароль).
2. Вкладка **«Опубликовать / редактировать»** → заполните форму, приложите файл.
3. Повторите для всех актов, которые хотите опубликовать. Все правки
   накапливаются в браузере.
4. Перейдите на вкладку **«Сохранить и опубликовать»** → кнопка
   **«⤓ Скачать архив для публикации (ZIP)»**.
5. Распакуйте ZIP в корень репозитория (он содержит `data/manifest.json` и
   при необходимости новые файлы в `data/acts/`).
6. ```bash
   git add data/manifest.json data/acts/
   git commit -m "Опубликование новых актов"
   git push
   ```
7. GitHub Pages обновит сайт через 1–2 минуты.

## Настройка

### Учётные данные админки

Файл `js/config.js`:

```js
window.ELOGOR_CONFIG = {
  ADMIN_PATH:     "bulbazavr-moldovan.html",
  ADMIN_LOGIN:    "admin",
  ADMIN_PASSWORD: "elogor-2025",
  // ...
};
```

Чтобы сменить «скрытый» URL: переименуйте сам HTML-файл (например,
в `mysecret-portal-1234.html`) и пропишите то же значение в `ADMIN_PATH`.

### Список типов актов

Хранится в `data/manifest.json` в массиве `types`. По умолчанию более 30
категорий. Если в админке нужно опубликовать акт вновь введённого типа —
просто добавьте элемент в массив `types` и закоммитьте.

### Атрибутика Республики

В `data/manifest.json` есть блок `republic` (название, девиз). Используется
в шапке портала.
